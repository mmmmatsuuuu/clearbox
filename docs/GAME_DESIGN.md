# ゲーム設計ドキュメント

## コンセプト

「授業で習った 16 進数が、なんの役に立つのかわからない」という感覚を解消するゲーム。  
プレイヤーはゲームを攻略するために自然とセーブデータを調べ、バイナリ表現を実感する。

## 対象プレイヤー

- 16 進数・2 進数の基礎を授業で習っている中学〜高校生
- Chromebook でプレイ
- プレイ時間の目安：全部わかっていれば 1 時間程度（詰まった場合の上限なし）

## ゲーム構成

### 舞台設定

機械王が頂上に住む自動機械の塔「大機関塔」を舞台にした、スチームパンク世界観の 2D ドット絵 RPG。  
プレイヤーの目的は塔を登り、最上階の機械王を倒して技師の姫を救うこと。  
世界観・キャラクター・ビジュアルの詳細は [WORLD_DESIGN.md](WORLD_DESIGN.md) を参照。  
※ 本ドキュメントの名称は新世界観に統一している。ゲーム内テキストへの反映はビジュアル実装フェーズで行うため、現行実装は旧名称のまま（対応表は WORLD_DESIGN.md）。

### フロア構成

| 階 | 内容 |
|---|---|
| -1F | 裏ボス（1F のからくり像を動かせば最初から挑める。勝つにはメギド = 機械王撃破が必要） |
| 1F | チュートリアル（NPC からゲームの仕組みとリングを受け取る） |
| 2F | ボス 1 |
| 3F | ボス 2 |
| 4F | ボス 3 |
| 5F | ボス 4 |
| 最上階 | 機械王 |

### 学習の段階設計

| 階 | 担当する概念 | 仕掛け | チート発覚の条件 |
|---|---|---|---|
| 2F | uint8 / 16 進数の基本 | HP を増やして生き残る | HP > 255 で発覚（「駆け出しの勇者が 255 超はあり得ない」） |
| 3F | 符号付き整数（int8） | 通れない座標（X = -1 = `0xFF`）の技を取得する | なし（罠として機能） |
| 4F | uint16 リトルエンディアン | HP の 2 バイト目を使って 255 超にしないと勝てない | なし（2F で「255 が上限」と思い込ませた上での発見） |
| 5F | NPC 交流コード | 1F NPC のコードを予測して NPC 交流スロットに設定 | なし |
| 最上階 | XOR チェックサム | リングの値が合わないと初期ステータスにリセットされて戦闘 | リングの不整合 → 初期ステータスにリセット（ゲームオーバーではない） |
| -1F | float32（後フェーズ） | 機械王から入手した技の威力を float32 として操作 | — |

### 2F → 4F の意図的な罠

2F で「HP が 255 を超えるとチートがバレる」という体験をさせることで、プレイヤーは HP の上限を 255 と思い込む。  
4F でどう頑張っても 255 HP では負ける状況を作り、「HP に 2 バイト目があるのでは？」という発見を促す。

### 3F の座標改ざん

通常は壁に囲まれてどう歩いても到達できない場所に技がある。  
座標は int8 で保存されており、X = -1 は `0xFF` として表現される。  
2F で「`0xFF` = 255（uint8）」と学んでいるため、「`0xFF` は int8 では -1 になる」という気づきが必要になる。

### 5F の機関登録番号

塔の住人には機関登録番号が振られており、会話ウィンドウの真鍮プレートに番号が刻印されて表示される。  
5F の祭壇は「最初の住人との絆」= 1F の老技師の番号を求める。誰が最初の住人かを思い出し、1F に戻ってプレートを読む必要がある。  
老技師の番号を NPC 交流スロット（NC）に書き込むと、その住人との絆を結んだ扱いになり、5F ボスに必要な技が手に入る。

### 機械王と誠実のリング

1F の老技師から「誠実のリング」を受け取る。  
リングの値はセーブデータの HP・スキル全 6 バイトの XOR チェックサム。正式名は「誠実の検算環」だが、チェックサムの存在を示唆しすぎないよう最上階まで明かさない。  
機械王だけがこの意味を知っており、リングの値が合わない場合は初期ステータスにリセットして戦闘を開始する。  
ゲームオーバーにはならないため、プレイヤーは弱い状態で機械王と戦うことになる。

## スキルシステム

- プレイヤーは最大 4 つのスキルを習得できる
- スキルは各フロアのダンジョンや NPC から入手する
- 入手時にスキルコード（1 バイト）が告知される（例：「ファイアのコード: `0xA3`」）
- セーブデータのスキルスロット（0x06〜0x09）に正しいコードをセットすることでスキルを装備する
- スキルの威力がそのままボスへのダメージになる（攻撃力というパラメータは存在しない）

## チート発覚の演出方針

- メタ的な発言はせず、没入感を維持する
- 発覚タイミングは NPC に話しかけたとき・ボス戦開始時
- 「ゲームオーバー」ではなく「自然な結果」として表現する

| 発覚条件 | セリフ例 | 結果 |
|---|---|---|
| HP > 255（2F ボス） | 「そなたの生命力…あり得ぬ数値だ」 | ゲームオーバー |
| リングの不整合（機械王） | 「誠実のリングが濁っておるぞ。貴様、不正をしておるな！」 | 初期ステータスにリセットして戦闘開始 |

## 演出設計（勝利・敗北・フィードバック）

テストプレイで「勝ったのかバグなのか分からない」「無反応で混乱する」事象が確認されたため、結果が曖昧になる場面すべてに明示的なフィードバックを定義する。

### 勝利演出（通常ボス: 2F〜5F）

現状は「倒した！」の直後にフィールドへ戻るだけで、勝利が伝わらない。以下のシーケンスに変更する。

1. とどめのダメージ表示 → ボスのスプライトが点滅して消滅（白フラッシュ 1 回）
2. 「（ボス名）を倒した！」
3. **勝利ウィンドウ**（金色の専用枠）：「勝利！　撃破の刻印がセーブデータに刻まれた。」 ← BD 領域への教育的な誘導を兼ねる
4. フィールド復帰後、1 回だけ自動メッセージ：「上へ続く階段が開いた！」（撃破で階段が解放される階のみ）

### 勝利演出（機械王）

1. 通常勝利シーケンス（既存の winLines を含む）
2. フィールド復帰後、姫が解放された状態で配置（既存実装どおり）
3. 自動メッセージ：「姫に話しかけてみよう。」 ← メギド解放（コード `0xFA`）の告知が姫のセリフにあるため、会話への誘導が必須

### 勝利演出（カオス = 真エンディング）

1. 既存の winLines（「真エンディング」）のあと画面を暗転
2. **クリア画面**：「ＣＬＥＡＲ！　すべての謎を解き明かした」＋ 学んだ概念の一覧（uint8 / int8 / リトルエンディアン / コード体系 / XOR チェックサム / float32）を表示
3. Z キーでタイトルへ戻る（セーブデータは保持したまま）

### 敗北演出（共通ゲームオーバー画面）

現状は「力尽きた...」→ 即タイトル画面でクラッシュに見える。以下に変更する。

1. 敗北メッセージ（下記）→ 画面を赤 → 黒にフェード
2. **ゲームオーバー画面**（黒背景の専用画面）：
   - 「ゲームオーバー」
   - 「冒険は振り出しに戻った。セーブデータがあるなら、ロードで再開できる。」 ← リセット仕様とロードの存在を必ず伝える
   - 「Z：タイトルへ」
3. Z キーでタイトルへ

| 敗北の種類 | 直前のメッセージ | 備考 |
|---|---|---|
| 通常敗北（HP 0） | 「力尽きた...」 | |
| チート発覚（2F・HP > 255） | 発覚セリフ（既存） | 同じゲームオーバー画面に合流 |
| 混沌の鏡による自滅（-1F） | 「自らの力に飲み込まれた…」（既存） | 同上 |

### セーブ・ロードのフィードバック

| 場面 | 現状 | 定義する演出 |
|---|---|---|
| セーブ成功 | ダウンロードのみ・無言 | 「セーブしました。save_data.txt を保存した。」 |
| ロード成功 | 無言で位置・HP が変わる | 「ロードしました。」（フロアが変わる場合はフェードイン後に表示） |
| **ロード失敗** | **完全に無反応（バグに見える）** | 「セーブデータが読み込めなかった。ファイルの形式を確認しよう。」 |

### その他の混乱ポイント

| 場面 | 現状 | 定義する演出 |
|---|---|---|
| ロック中の階段（ボス未撃破）を踏む | 無反応 | 「階段は固く閉ざされている。この階の主を倒す必要がありそうだ。」（1 回踏むごとに表示） |
| 1F のからくり像を動かして隠し階段が出現 | 無言で階段が現れる | カメラシェイク（小）＋「ゴゴゴ…どこかで床の開く音がした。」 |
| 3F の隠し部屋（CX = -1）に入った瞬間 | 通常移動と同じ | 画面端フラッシュ＋「壁の向こう側に踏み込んだ…！」（座標改ざん成功の確認） |

### 実装上の注意

- メッセージはすべて既存の `DialogBox` を流用する（新規 UI は勝利ウィンドウ・ゲームオーバー画面・クリア画面の 3 つ）
- ゲームオーバー画面とロード失敗の通知は誤認リスクが最も高いため、**ビジュアル実装より先に**単体で実装してよい
- 効果音・BGM は未決定事項のまま（演出はすべて視覚+テキストで成立させる）

## マップシステム

- 各フロアのマップは最大 **16×16 タイル**
- 画面には勇者を中心に **5×5 タイル**（canvas: 320×320px、タイルサイズ: 64px）のみ表示
- カメラが勇者を追従し、移動で全体を把握する探索型レイアウト

## ゲームオーバー

- 勇者 HP が 0 になった場合、またはチート発覚でゲームオーバーになった場合
- `SaveManager.reset()` で全ステートを初期化しタイトルへ戻る
- ロードで再開できるため、セーブの重要性を体験させる

### 2F NPC

| 配置 | セリフ |
|---|---|
| 入口付近 | 「ここのボスは粘り強いらしいぞ！頑張れよ！」 |
| 中盤 | 「おまえそんな HP で大丈夫か？」 |

- 最低 HP 16 以上で理論上撃破可能（余裕を持つなら 30 前後）

## 3F〜-1F 設計

### ボス一覧

| 階 | ボス | HP | 攻撃力 | 再生/ターン | 撃破コード | 攻略の鍵 |
|---|---|---|---|---|---|---|
| 3F | 蒸気ゴーレム | 60 | 20 | 10 | `0x29` | 斬撃(10)では再生に追いつかない。隠し部屋のファイア(40)が必要 |
| 4F | からくりデュラハン | 300 | 90 | — | `0xD4` | サンダー(75)でも撃破まで270被弾。HP 255 では必ず負ける |
| 5F | 機竜 | 240 | 50 | 80 | `0x86` | サンダー(75)は再生(80)に届かない。覇王斬(120)が必要 |
| 最上階 | 機械王 | 3000 | 2000 | 100 | `0x4B` | リング不整合なら初期ステータスにリセットされ戦闘。ホーリー(200)で約30ターン、被ダメージ約58000 → HP を uint16 の限界（65535）近くまで使い切る必要がある |
| -1F | カオス | 50000 | 500 | 500 | —（記録なし） | メギド初期値(250)は再生(500)に届かない。float32 の書き換えと「混沌の鏡」の攻略が必要 |

### -1F の「混沌の鏡」ギミック

カオスは受けたダメージと同じダメージをプレイヤーに跳ね返す。

- float32 は約 3.4×10^38 まで表現できるが、最大値にすると反射で自分が即死する
- 勝利条件：メギド威力 > 50000（HP を一撃で抜く）かつ 威力 < 自分の HP（最大 65535）
- 有効な威力の窓は **50001〜65534 程度**。float32 で狙った値（例：60000 = `00 60 6A 47`）を作る必要がある
- 「大きければ大きいほど良い」という直感を裏切り、float32 の値設計の理解を試す仕掛け
- -1F の NPC が鏡の存在と「過ぎたる力は身を滅ぼす」ことを警告する

### スキル一覧

| スキル | コード | 威力 | 入手 |
|---|---|---|---|
| 斬撃 | `0xB3` | 10 | 初期装備 |
| 防御 | `0x5F` | —（1ターン無効化） | 2F 東の通路 |
| ファイア | `0xA3` | 40 | 3F 隠し部屋（CX = -1 = `0xFF`） |
| サンダー | `0xC7` | 75 | 4F 北東の小部屋 |
| 覇王斬 | `0xE4` | 120 | 5F の刻印の祭壇（NC スロットに 1F 老技師の登録番号 `0x9D` が必要） |
| ホーリー | `0xD9` | 200 | 最上階の祭壇横 |
| メギド | `0xFA` | float32（初期 250.0） | 機械王撃破で解放。威力は RVx16–x19 に float32 LE で保存 |

スロットは 4 つしかないため、5 つ目以降のスキルはセーブデータのスキルスロット書き換えで入れ替える（リングの再計算が必要になる）。

### 機関登録番号（5F の謎解き）

- 住人・魔物の登録番号は撃破コードと同様、実装時にランダムに割り当てた固定値（規則性なし）
- 各 NPC の番号は会話ウィンドウの真鍮プレートに刻印されて表示される（NPC は番号を口にしない）
- 5F の刻印の祭壇は「最初の住人との絆」を求める。1F の老技師のプレートを読み、その番号 `0x9D` を NC スロットに書き込むと覇王斬のコードが告知される

### -1F の解放

- 1F の機械王のからくり像を動かすと隠し階段が現れ、最初から -1F に降りられる
- ただしカオスに勝つにはメギド（機械王撃破で解放）の float32 書き換えが必須。序盤に挑むと「全然かなわない」体験になる
- -1F の NPC が float32 のビット構成（符号1・指数8・仮数23）と「250 = `00 00 7A 43`」を語る

## UI / UX

- ドット絵スタイル（スチームパンク × 旧ドラクエ風。カラーパレット等は [WORLD_DESIGN.md](WORLD_DESIGN.md) を参照）
- 操作：キーボード（矢印 + Z / X）または画面タップ（Chromebook 対応）
- セーブ：ボタンクリックで `save_data.txt` をダウンロード
- ロード：ファイルアップロードボタンで `.txt` を選択

### ステータス画面

```
┌───────────────────────┐
│ HP  : 10              │
│ リング: ✦ 光っている  │  ← チェックサムが正常
│ スキル 1: 斬撃        │
└───────────────────────┘
```

ステータス画面を開くたびにチェックサムを計算し、リングの値と照合する。

| 状態 | 表示 | 意味 |
|---|---|---|
| チェックサム一致 | ✦ 光っている | セーブデータの整合性が保たれている |
| チェックサム不一致 | ◆ くすんでいる | どこかのフィールドがリングと合っていない |

リングの値をステータス画面で直接表示しないことで、「リングとは何か」という謎を機械王戦まで持続させる。

## セーブ・ロードの UX フロー

```
[セーブ]
プレイヤーが「セーブ」を選択
→ save_data.txt をブラウザからダウンロード
→ "セーブしました" メッセージ

[ロード]
タイトル画面の「ロード」ボタン
→ ファイル選択ダイアログ
→ ファイルを読み込み・検証（マジックバイト・バイト数）
→ ゲーム再開
```

## ヒントの設計方針

- NPC・看板などで「セーブデータのここを調べると…」レベルのヒントを出す
- 答えをそのまま教えるのではなく「どのオフセットを見ればいいか」程度にとどめる
- ヒントの量・質はテストプレイで調整する
