# ゲーム設計ドキュメント

> **ステータス**: フェーズ 2（最初のステージ実装）以降のテストプレイで更新予定

## コンセプト

「授業で習った 16 進数が、なんの役に立つのかわからない」という感覚を解消するゲーム。  
プレイヤーはゲームを攻略するために自然とセーブデータを調べ、バイナリ表現を実感する。

## 対象プレイヤー

- 16 進数・2 進数の基礎を授業で習っている中学〜高校生
- Chromebook でプレイ
- プレイ時間の目安：1〜2 時間

## ゲーム構成

### 規模

- ボス 4 体（最終的な体数はテストプレイで調整）
- 1 ボスにつき 1 つの「データ操作の概念」を担当
- マップは最小限（学習の邪魔をしない）

### 学習の段階設計

| ボス | 担当する概念 | 難易度 |
|---|---|---|
| ボス 1 | 16 進数の基本・バイトの読み書き | ★☆☆☆ |
| ボス 2 | （テストプレイ後に決定） | ★★☆☆ |
| ボス 3 | （テストプレイ後に決定） | ★★★☆ |
| ボス 4 | （テストプレイ後に決定） | ★★★★ |

候補となる概念：ビットフラグ / リトルエンディアン / unsigned int の最大値 / XOR チェックサム

### ゲームの流れ

1. ゲーム開始 → 村でストーリー説明
2. ダンジョンへ → ボス戦（普通に戦うと負ける）
3. 敗北後「セーブして出直そう」というヒント
4. セーブデータを書き換えて再挑戦 → クリア
5. 次のダンジョンへ（繰り返し）

### ヒントの設計方針

- **ゲーム内ヒント**：NPC・看板などで「セーブデータのここを調べると…」レベルのヒントを出す
- **ヒントの量はテストプレイで調整**する
- 答えをそのまま教えるのではなく「どのオフセットを見ればいいか」程度にとどめる

## UI / UX

- ドット絵スタイル（旧ドラクエ風）
- 操作：キーボード（矢印 + Z / X）または画面タップ（Chromebook 対応）
- セーブ：ボタンクリックで `.txt` ファイルをダウンロード
- ロード：ファイルアップロードボタンで `.txt` を選択

## セーブ・ロードの UX フロー

```
[セーブ]
プレイヤーが「セーブ」を選択
→ save_data.txt をブラウザからダウンロード
→ "セーブしました" メッセージ

[ロード]
タイトル画面の「ロード」ボタン
→ ファイル選択ダイアログ
→ ファイルを読み込み・検証
→ ゲーム再開（検証失敗時はエラーメッセージ）
```
