# セーブデータ仕様

## ファイル形式

- **拡張子**: `.txt`
- **文字コード**: ASCII
- **内容**: 16 進数をスペース区切りで並べたテキスト
- **サイズ**: 32 バイト固定（2 行）

### 表示形式

```
43 42 01 0A 00 09 05 00 00 00 02 05 01 00 00 00
00 00 00 00 00 00 00 00 00 00 00 00 00 00 00 00
```

1 行 16 バイト（32 文字 + スペース）で改行する。  
ヘックスエディタの一般的な表示と同じレイアウト。

## 設計方針

- 難読化・暗号化は**行わない**
- フォーマットはシンプルに保ち、テキストエディタで誰でも開ける
- **ステータス画面の表示順とセーブデータのフィールド順を一致させる**（プレイヤーが対応に気づきやすくなる）

## フィールド一覧

| オフセット | サイズ | 型 | フィールド | ステータス画面 | 関連フロア |
|---|---|---|---|---|---|
| 0x00–0x01 | 2B | ASCII | マジック `CB` | — | — |
| 0x02 | 1B | uint8 | バージョン（`0x01`） | — | — |
| 0x03–0x04 | 2B | uint16 LE | HP | **表示** | 2F・4F |
| 0x05 | 1B | uint8 | 護符（XOR チェックサム） | **✦/◆ で表示** | 最上階 |
| 0x06–0x09 | 4B | uint8 × 4 | スキルスロット 0〜3（コード） | **表示** | 3F・5F |
| 0x0A | 1B | int8 | 勇者 X 座標 | 非表示 | 3F |
| 0x0B | 1B | int8 | 勇者 Y 座標 | 非表示 | 3F |
| 0x0C | 1B | int8 | 勇者 Z 座標（階数） | 非表示 | — |
| 0x0D–0x0F | 3B | uint8 × 3 | NPC 交流コード 0〜2 | 非表示 | 5F |
| 0x10–0x14 | 5B | uint8 × 5 | ボス撃破コード（2F〜魔王） | 非表示 | — |
| 0x15–0x18 | 4B | — | 予約 | — | — |
| 0x19–0x1C | 4B | float32 | 予約（v2: 特殊スキル威力） | — | -1F |
| 0x1D–0x1F | 3B | — | 予約 | — | — |

## フィールドの詳細

### HP（0x03–0x04）

- 型：uint16 リトルエンディアン
- 例：HP = 300 → `2C 01`（0x012C = 300）
- 2F ボスは HP > 255 を検出する（「駆け出しの勇者が 255 超はあり得ない」）
- 4F では 255 超の HP が必要になり、2 バイト目の存在に気づかせる設計

### 護符（0x05）

- 型：uint8
- ステータス画面では値を直接表示せず「✦ 光っている / ◆ くすんでいる」で表示
- ステータス画面を開くたびに計算し、保存値と照合して表示を更新する
- セーブデータ上は 1 バイトの 16 進数として存在するため、プレイヤーが改ざんできる
- 魔王だけが護符の意味を知っており、不整合の場合は初期ステータスにリセットして戦闘を開始する

#### チェックサムの計算範囲

```
護符（0x05）= XOR of 0x03–0x04 and 0x06–0x14
```

対象：HP・スキルスロット・XYZ 座標・NPC 交流コード・ボス撃破コード  
対象外：護符自身（0x05）・予約バイト（0x15 以降）

#### 計算例（初期状態）

```
0x0A XOR 0x00           （HP: 10）
XOR 0x05                （スキル0: 斬撃 コード0x05）
XOR 0x00 XOR 0x00 XOR 0x00  （スキル1〜3: なし）
XOR 0x02 XOR 0x05       （X=2, Y=5）
XOR 0x01                （Z=1, 1F）
XOR 0x00 × 8            （NPC・ボス: 全て未取得）
= 0x09

護符 = 0x09
```

### スキルスロット（0x06–0x09）

- 型：uint8 × 4（各スロット 1 バイト）
- 値 `0x00` = 空きスロット
- スキルコードがそのまま威力（ダメージ）になる（例：コード `0x05` = 威力 5）
- スキル取得時にコードが告知される（例：「ファイア スキルコード: `0xA3`」）
- 5 つ以上のスロットを使おうとすると不正扱い
- ゲーム開始時はスロット 0 に斬撃（コード `0x05`）がセットされている

### 座標（0x0A–0x0C）

- 型：int8（符号付き 1 バイト整数）
- X・Y：フロア内のタイル座標
- Z：階数（1F = `0x01`、-1F = `0xFF`）
- 3F の隠し部屋は X = -1（`0xFF`）に位置する

### NPC 交流コード（0x0D–0x0F）

- 型：uint8 × 3
- 値 `0x00` = 交流なし
- 5F の攻略には 1F NPC のコードをこのスロットに入れる必要がある
- NPC コードの規則性は敵撃破コードのパターン（連番）から推測できる

### ボス撃破コード（0x10–0x14）

| オフセット | ボス |
|---|---|
| 0x10 | 2F ボス |
| 0x11 | 3F ボス |
| 0x12 | 4F ボス |
| 0x13 | 5F ボス |
| 0x14 | 魔王 |

- 値 `0x00` = 未撃破
- 正規に倒したときのみ正しいコードが書き込まれる
- 間違ったコードを入れても次のフロアに進めない（扉が開かない）

## 初期状態（ゲーム開始時）

```
43 42 01 0A 00 09 05 00 00 00 02 05 01 00 00 00
00 00 00 00 00 00 00 00 00 00 00 00 00 00 00 00
```

| フィールド | 初期値 | 備考 |
|---|---|---|
| HP | `0A 00`（10） | 弱い状態でゲーム開始 |
| 護符 | `09` | HP・スキル・XYZ の XOR 結果 |
| スキル0 | `05` | 斬撃（コード = 威力） |
| スキル1〜3 | `00 00 00` | 全スロット空き |
| X, Y | `02 05` | 1F の初期位置 |
| Z | `01` | 1F |
| その他 | `00` | 全て未取得・未撃破 |

## 実装メモ

- セーブ処理：ゲーム状態 → バイト列 → 16 進テキスト → `Blob` → ダウンロード
- ロード処理：ファイル読み込み → テキスト解析 → バイト列 → ゲーム状態復元
- バリデーション：マジックバイト確認・バイト数（32 バイト）チェック
- 護符の検証はステータス画面表示時および魔王戦開始時に実施する（ロード時には検証しない）
