# セーブデータ仕様

> **ステータス**: フェーズ 3（テストプレイ後）に詳細を決定予定  
> このドキュメントは現時点での方針と設計指針を記載する

## ファイル形式

- **拡張子**: `.txt`
- **文字コード**: ASCII
- **内容**: 16 進数をスペース区切りで並べたテキスト

### 表示形式の例

```
43 4C 01 A3 64 00 64 00 00 00 00 00 E8 03 01 00
48 65 72 6F 00 00 00 00 00 00 00 00 00 00 00 00
```

1 行 16 バイト（32 文字 + スペース）で改行する。  
これはヘックスエディタの一般的な表示と同じレイアウトで、授業で習った形式と対応しやすい。

## 設計方針

### 教育目的を最優先する

- 難読化・暗号化は**行わない**
- フォーマットはシンプルに保ち、テキストエディタで誰でも開ける
- 各フィールドの意味はゲーム内ヒントや授業資料と対応させる

### バイト列の構造（詳細は TBD）

以下は現時点の草案。テストプレイ後に確定する。

| オフセット | サイズ | 型 | 説明 |
|---|---|---|---|
| 0x00 | 2B | ASCII | マジックバイト（ファイル識別子） |
| 0x02 | 1B | uint8 | バージョン番号 |
| 0x03 | 1B | uint8 | チェックサム（採用可否は TBD） |
| 0x04 〜 | TBD | TBD | ゲームデータ（フェーズ 3 で確定） |

### 学習する概念とフィールドの対応（草案）

| 学習概念 | フィールドの種類 |
|---|---|
| 16 進数の基本 | HP 値（1〜2 バイト整数） |
| ビットフラグ | 所持アイテム・フラグ群（1 バイト） |
| リトルエンディアン | 大きな数値（2〜4 バイト整数） |
| チェックサム | ファイル整合性チェック（TBD） |

## 実装メモ

- セーブ処理：ゲーム状態 → バイト列 → 16 進テキスト → `Blob` → ダウンロード
- ロード処理：ファイル読み込み → テキスト解析 → バイト列 → ゲーム状態復元
- バリデーション：マジックバイトの確認、バイト数のチェック
